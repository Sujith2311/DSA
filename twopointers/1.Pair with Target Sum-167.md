# Intuition
Since the array is already sorted, for every element we can figure out exactly what value we still need to reach the target (`target - numbers[i]`). Instead of checking every other element linearly, we can binary search for that needed value in the remaining sorted portion of the array.

# Approach
- Iterate through the array with index `i` from `0` to `n-1`, treating `numbers[i]` as the first number of the pair.
- Compute `need = target - numbers[i]`, the value the second number must equal.
- Binary search for `need` in the subarray to the right of `i` (from `i+1` to `n-1`), since the array is sorted and we want the second index to come after the first.
- If `numbers[mid] == need`, we found our pair — return `[i+1, mid+1]` (1-indexed as required).
- If `numbers[mid] < need`, search the right half (`left = mid+1`).
- If `numbers[mid] > need`, search the left half (`right = mid-1`).
- Since the problem guarantees exactly one solution, the loop will always return before finishing.

# Complexity
- Time complexity:
$$O(n \log n)$$
For each of the `n` elements, we perform a binary search over up to `n` elements, costing `O(log n)`.

- Space complexity:
$$O(1)$$
Only a constant amount of extra space is used (pointers and variables) — no additional data structures.

# Code
```python []
class Solution(object):
    def twoSum(self, numbers, target):
        n=len(numbers)
        for i in range(n):
            need=target-numbers[i]
            left=i+1
            right=n-1
            while left<=right:
                mid=(left+right)//2
                if numbers[mid]==need:
                    return [i+1,mid+1]
                elif numbers[mid]<need:
                    left=mid+1
                else:
                    right=mid-1
```
